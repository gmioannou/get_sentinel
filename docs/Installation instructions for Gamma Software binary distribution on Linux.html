These instructions apply for Linux OS, (e.g. 64-bit Ubuntu 22.04 LTS). The Gamma Software is for 64-bit operating systems.

Installing the software requires the following steps:

  1. Expanding of the Gamma Software gzipped tar file and setup environment variables
    1. Unpack the Gamma Software archive
    2. Set Environment variables
    3. Select and Set Default Raster Image Format
  2. Install the FFTW3 libraries for data that are 32-bit floating point numbers (libfftw3f.so)
  3. Install Gnuplot and GIMP programs
  4. Install the GDAL library (required for TerraSAR-X PRI and geocoded data)
  5. Install the HDF5 library (required for COSMO/SKYMED and ICEYE data)
  6. Install the NETCDF library (required for SWOT data)
  7. Install LAPACK and BLAS libraries (required for polarimetric programs in the LAT)
  8. Test the software installation
  9. Gamma Software documentation
  10. Install tcsh, perl
  11. Install Python 3
  12. Python and Matlab wrappers
    Note: logging into a Linux server containing the Gamma Software from Windows

As a very first step, make sure your Linux system is up-to-date. Use the following command to check and update your system:

Ubuntu:
sudo apt update
sudo apt upgrade

CentOS, RHEL, Rocky Linux:
sudo yum check-update
sudo yum update

RHEL9/10, Rocky Linux 9/10:
sudo dnf check-update
sudo dnf update
sudo dnf config-manager --set-enabled crb
sudo dnf install epel-release
sudo crb enable

1. Expanding of the Gamma Software package (gzipped tar format, *.tar.gz)  and setup environment variables

1.1 Unpack the Gamma Software software archive

Select the directory where you would like to unpack the Gamma Software. The suggested location is your home directory or /usr/local:

export GAMMA=/usr/local

or

export GAMMA=$HOME

The software is delivered as a gzipped tar file. Copy the downloaded file to the $GAMMA_HOME directory:

cp GAMMA_SOFTWARE-yyyymmdd.linux.tar.gz $GAMMA
cd $GAMMA_HOME

Unpack the distribution using gunzip and tar:
gunzip -c GAMMA_SOFTWARE-yyyymmdd.linux.tar.gz > GAMMA_SOFTWARE-yyyymmdd.linux.tar
tar -xvf GAMMA_SOFTWARE-yyyymmdd.linux.tar

1.2 Set Environment Variables

The default shell to run the software is bash. The environment variables for compilation and running the software can be set in the .bashrc file located in your home directory.

export GAMMA_HOME=/usr/local/GAMMA_SOFTWARE-yyyymmdd
#export GAMMA_HOME=$HOME/GAMMA_SOFTWARE-yyyymmdd
export ISP_HOME=$GAMMA_HOME/ISP
export DIFF_HOME=$GAMMA_HOME/DIFF
export DISP_HOME=$GAMMA_HOME/DISP
export LAT_HOME=$GAMMA_HOME/LAT
export IPTA_HOME=$GAMMA_HOME/IPTA
export GEO_HOME=$GAMMA_HOME/GEO

export PATH=$PATH:.:$MSP_HOME/bin:$ISP_HOME/bin:$DIFF_HOME/bin:$LAT_HOME/bin:$IPTA_HOME/bin:$GEO_HOME/bin:\
$DISP_HOME/bin:$MSP_HOME/scripts:$ISP_HOME/scripts:$DIFF_HOME/scripts:$LAT_HOME/scripts:$IPTA_HOME/scripts:\
$GEO_HOME/scripts:$DISP_HOME/scripts

export OS=linux64

#This adds the Gamma Software main directory to the Python path
export PYTHONPATH=.:$GAMMA_HOME:$PYTHONPATH

#This command disables version checking for HDF5
export HDF5_DISABLE_VERSION_CHECK=1

#Alternately, this sets the default terminal type to be WXT
export GNUTERM=wx

#set default raster format to BMP
export GAMMA_RASTER="BMP"

1.3 Select and Set Default Raster Image Format

The default raster image format can be set. By default it is bitmap (BMP) on all platforms. The environment variable GAMMA_RASTER can be set to change the default format for your installation:

SUN_RASTER (file extension: ras)
BMP (file extension: bmp)
TIFF (file extension: tif)

To select BMP as the default raster format, then enter:

export GAMMA_RASTER="BMP"
export GAMMA_RASTER="TIFF"

will set the default format to TIFF

export GAMMA_RASTER="SUN_RASTER"

will set the default format to Sun raster (not recommended)
To make the command permanent, add the export command to your .bashrc file.

2. Install the FFTW3 libraries

This library of FFT routines is required for the Gamma Software to run and must be installed on your system. The libraries come in 2 forms, either for float (4 byte, 32-bit) or double (64 bit) floating point numbers. Gamma software uses the version that works with 32-bit floating point numbers (single).

Debian based Linux distributions (Debian, Ubuntu ... have packages for FFTW3) up to Ubuntu 22.04:
sudo apt install libfftw3-3 libfftw3-dev libfftw3-single3

Ubuntu 24.04:
sudo apt install libfftw3-bin libfftw3-dev libfftw3-single3

Redhat based Linux distributions (RHEL, Fedora, CentOS have FFTW3 RPM packages:
yum search fftw
sudo yum install fftw-devel

Alternately, you can also download FFTW3 source code from the FFTW3 server:
http://www.fftw.org/download.html

The source code file is:
fftw-3.3.4.tar.gz

Use the following configure options to build the software:
./configure --disable-fortran --enable-single --enable-shared --enable-sse --enable-sse2

To build and install the software:
make
sudo make install

3. Install Gnuplot and GIMP programs

Gnuplot on Linux (Ubuntu, Debian) now uses the QT and WXT widget sets by default. We recommend using QT on Linux as in our experience it draws the plots faster and refreshes them more reliably than WXT:

Ubuntu:
sudo apt install gnuplot gnuplot-qt gnuplot-data gimp gnome-icon-theme

To test your installation of gnuplot, start the program and plot sin(x) in a pop-up window: add this to your .bashrc file:
export GNUTERM=qt

and test the installation:
gnuplot
plot sin(x)

If your Linux distribution is RHEL7, CentOS7, only the X11 terminal is supported:
sudo yum install gnuplot gimp gnome-icon-theme

On Rocky Linux 8 / RHEL8:
sudo dnf install gnuplot gimp

On Rocky Linux 9 / RHEL9:
sudo dnf install gnuplot gimp gnome-icon-theme

On Rocky Linux 10, Gnuplot and GIMP are not (yet) available from "dnf". Gnuplot is required for some Gamma programs, it can be installed by compiling it from the source code:

Install the following libraries:
sudo dnf install libX11-devel libXft-devel cairo-devel pango-devel freetype-devel libpng-devel \
libjpeg-turbo-devel readline-devel libcerf-devel qt6-qtbase-devel qt6-qtdeclarative-devel qt6-qtsvg-devel \
qt6-qttools-devel qt6-qt5compat-devel qt6-qtbase-gui

Download the latest stable version of Gnuplot:
wget https://sourceforge.net/projects/gnuplot/files/gnuplot/6.0.3/gnuplot-6.0.3.tar.gz

Extract the Gnuplot archive and enter in the main directory:
tar xvf gnuplot-6.0.3.tar.gz
cd gnuplot-6.0.3/

Compile (check for warnings when running "configure", you may need to install additional libraries):
./configure --prefix=/usr/local --with-readline=gnu --with-cairo
make
sudo make install

add this to your .bashrc
export GNUTERM=qt #or GNUTERM=x11 for RHEL7

to test your gnuplot installation:
gnuplot
plot sin(x)

CentOS/RHEL 7:

If your Linux distribution is RHEL7, CentOS7, the Verdana font needs to be installed. This can be done by installing Microsoft fonts:
sudo yum install curl cabextract xorg-x11-font-utils fontconfig
sudo yum install https://downloads.sourceforge.net/project/mscorefonts2/rpms/msttcore-fonts-installer-2.6-1.noarch.rpm

Create a soft link for the Verdana font:
cd /usr/share/fonts/msttcore
sudo ln -s verdana.ttf Verdana.ttf

Rebuild the fonts cache (may not be necessary):
sudo fc-cache -f -v

Add following line in the .bashrc file:
export GDFONTPATH=/usr/share/fonts/msttcore

4. Install the GDAL library

Ubuntu: sudo apt install gdal-bin libgdal-dev
Rocky Linux 8, RHEL8: sudo yum install gdal gdal-devel
Rocky Linux 9/10, RHEL9/10: sudo dnf install gdal gdal-devel

GDAL is available as a precompiled package for most Linux distributions and can be installed using package manager on your system. GDAL not only features a data abstraction library but has also powerful tools to convert geospatial data formats. The home web page for GDAL is:
http://www.gdal.org/

If you get an error message using one of these programs that indicates a missing library, but you have installed GDAL, the reason can be that the name of the library is not the same that was used at the time the program was built. This can be checked with the Linux command ldd. prints out all the required libraries and their expected location:
ldd $DIFF_HOME/bin/par_TX_geo

CentOS/RHEL 7:

Some Gamma Software programs now require a GDAL version newer than the version provided in CentOS 7 standard repository. Hence, newer versions of GDAL and PROJ have to be installed manually as follows:

Remove GDAL version from standard repository:
sudo yum remove gdal gdal-devel

Install libraries required for installing PROJ and GDAL:
sudo yum install sqlite-devel libzstd-devel libwebp-devel

Install PROJ:
cd ~/Downloads
wget https://download.osgeo.org/proj/proj-6.1.1.tar.gz
tar xf proj-6.1.1.tar.gz
cd proj-6.1.1
./configure
make
sudo make install

Install GDAL:
wget https://download.osgeo.org/gdal/2.4.2/gdal-2.4.2.tar.gz
tar xf gdal-2.4.2.tar.gz
cd gdal-2.4.2
./configure --with-proj
make
sudo make install

Add the following line to your .bashrc file:
export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:/usr/local/lib

5. Install the HDF5 library (required for COSMO/SKYMED and ICEYE data)

Ubuntu 18.04: sudo apt install libhdf5-dev libhdf5-100
Ubuntu 20.04 / 22.04: sudo apt install libhdf5-dev libhdf5-103
Ubuntu 24.04: sudo apt install libhdf5-dev libhdf5-103-1t64
CentOS 7, Rocky Linux 8, RHEL7/8: sudo yum install hdf5 hdf5-devel
Rocky Linux 9/10, RHEL9/10: sudo dnf install hdf5 hdf5-devel

HDF5 is needed for par_CS*. This program reads Cosmo/Skymed data. It is also needed for par_ICEYE_SLC. The HDF5 binary library is available as a precompiled package for most Linux distributions and can be installed using the package manager on your system.

The web page for HDF5 is:
https://portal.hdfgroup.org/

There are several different variants of the HDF5 library. Be sure to install the serial version. If you get an error message using one of these programs that indicates a missing library, but you have installed libhdf5, the reason can be that the name of the library is not the same as at the time the program was built. This can be checked with the Linux command ldd:
ldd $DIFF_HOME/bin/par_CS_geo

6. Install the NETCDF library (required for SWOT data)

Ubuntu: sudo apt install libnetcdf-dev
CentOS, RHEL: sudo yum install netcdf netcdf-devel

NETCDF is needed for par_SWOT_SLC. This program reads SWOT data. The NETCDF binary library is available as a precompiled package for most Linux distributions and can be installed using the package manager on your system.

The web page for NETCDF is:
https://www.unidata.ucar.edu/software/netcdf/

7. Install LAPACK and BLAS libraries (for Gamma Software LAT package only)

Ubuntu: sudo apt install libblas-dev libblas3 liblapack-dev liblapack3 liblapack-doc
CentOS, RHEL: sudo yum install blas blas-devel lapack lapack-devel

All major Linux distributions include the LAPACK and BLAS Libraries as prebuilt packages that can be downloaded and installed. These libraries are used for linear algebra calculations. Be sure to also install the developer packages (*-dev) since you are compiling programs with lapack.

8. Test the software installation

For a quick test to check that the search path is correctly set for the Gamma Software and libraries enter:
disras

The command line options for this program should then be displayed in the terminal window.

Another test can be to open the Gamma Software documentation using the gamma_doc Perl script. Enter for example:
gamma_doc data2geotiff

The documentation related to data2geotiff should open up in the web browser.

9. Gamma Software documentation

The Gamma Software documentation browser is an HTML-based system for viewing the web pages and PDF documents. The documentation browser includes for each module a Contents sidebar on the right side of the screen and a search functionality (within the module). You can access the documentation by opening Gamma_documentation.html (located in the main Gamma Software directory) in your web browser.

The program gamma_doc facilitates the access to the documentation related to a given module or program:

gamma_doc Opens the main page of the Gamma documentation browser.
gamma_doc DIFF Opens the DIFF/GEO documentation.
gamma_doc gc_map2 Opens the reference manual web page for gc_map2.

10. Install tcsh, perl

Ubuntu: sudo apt install tcsh perl
CentOS RHEL: sudo yum install tcsh perl

11. Installation of Python 3

The Gamma Software includes an increasing number of Python scripts, such as the Python display tools in the $DISP_HOME/scripts. Our scripts use Python modules that may not be installed per default, they will need to be added. They include the following modules: scipy, numpy, matplotlib, etc. When a module is missing, the Python script that needs it will throw an error message that includes the missing module name. Simply install that module in the same way as described below.

We recommend using Python 3, although our scripts have all been successfully tested using Python 2.

In general, we recommend using the Python 3 version provided as part of your Linux installation or from standard repositories:

Ubuntu:

Usually Python is already installed per default on Ubuntu. Check your version using:

python --version

If needed, use the following commands to install Python 3:

sudo apt update
sudo apt install python3

If needed/desired, set Python 3 as your default Python version as follows:

sudo apt install python-is-python3

Install additional modules as follows:

sudo apt install python3-numpy python3-matplotlib python3-scipy python3-shapely python3-packaging python3-ipython ...

CentOS7, RHEL7:

Install Python 3 and additional modules as follows:

sudo yum install python3 python3-numpy python-matplotlib scipy python3-shapely python3-packaging ...

Rocky Linux 8, 9, 10, RHEL8, 9, 10:

Install Python 3 and additional modules as follows:

sudo dnf install python3 python3-numpy python3-matplotlib python3-scipy python3-shapely python3-packaging python3-ipython ...

You may want to add a soft link so that by calling python you automatically call python3:

sudo ln -s /usr/bin/python3 /usr/local/bin/python

Alternate installation of Python

Other possibilities include installing Conda-Forge or Anaconda Python. These installations generally work well, but they may cause conflicts for binary programs that use the GDAL library. In that case, you can temporarily deactivate your Conda environment using the command conda deactivate. The Conda environment can be reactivated using conda activate. Another possibility is to remove / not use the GDAL package from your Conda Python installation.

To install the latest Conda-Forge Python version, download the text-based installer Miniforge3-Linux-x86_64.sh from https://github.com/conda-forge/miniforge. Then run the installer from the command-line:
bash Miniforge3-Linux-x86_64.sh

Install these recommended packages:
conda install scipy matplotlib numpy pandas pyserial pillow oct2py pytz colorspacious shapely
 conda install seaborn cmocean colorcet

The Miniforge installer will add a section at the end of your .bashrc that modifies the path so that the Conda-Forge Python distribution is at the start of the path ensuring that this is the Python version that runs. It is recommended to use #!/usr/bin/env python as the first line in any script so that the path is searched for the first instance of Python and that interpreter is used for the program.

12. Python and Matlab wrappers

The Gamma Software is integrated into Python through wrapper: the py_gamma module. It permits a smooth usage of the Gamma Software within Python scripts, Jupyter Notebooks, as well as within a Python Interactive Development Environment (IDE) such as Spyder or PyCharm. More information is available in py_gamma_user_guide.pdf (located in the main Gamma Software directory).

Similarly, the Gamma Software is integrated into Matlab through a wrapper composed of the mat_gamma and par_file classes. It permits a smooth usage of the Gamma Software within an interactive use of Matlab as well as within Matlab scripts. More information is available in mat_gamma_user_guide.pdf (located in the main Gamma Software directory).

Note: logging from Windows to a Linux server containing the Gamma Software

On the Windows machine:
- Install Xming server and Xming-fonts: https://sourceforge.net/projects/xming/files/
- Install MSYS2 (see instructions in INSTALL_win64.html)
- Be sure ssh tools for MSYS2 have been installed
- Add the following line in the MSYS2 .bashrc:
 export DISPLAY=localhost:0
- Edit /etc/ssh/ssh_config as follows:
  ForwardX11 yes
 ForwardX11Trusted yes

You should now be able to access the Linux server using the following command:
ssh -X user@server

if the display fails, try the -Y option:
ssh -Y user@server

You should now be able to use the Linux server.

As an alternative, install WSL on the Windows machine (see instructions in INSTALL_wsl.html), install a Linux version on WSL, and use it to access the Linux server.
